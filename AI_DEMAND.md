# 贪吃蛇游戏需求文档

## 项目概述

开发一个功能完整的贪吃蛇游戏网站，包含速度选择、障碍物系统和特殊食物等特性。

## 核心功能需求

### 1. 速度选择系统（游戏开始前）

- **界面要求**：游戏启动后显示5个速度选项按钮
- **速度档位**：
  - 慢速：200ms/格
  - 中速：150ms/格
  - 快速：100ms/格
  - 超速：70ms/格
  - 极速：40ms/格
- **交互逻辑**：
  - 点击速度按钮后高亮显示选中状态
  - 点击"确认速度"按钮后进入游戏界面
  - 速度一旦选定，游戏全程保持不变

### 2. 游戏界面

- **布局结构**：
  - 速度选择界面（初始显示）
  - 游戏主界面（速度确认后显示）
- **元素组成**：
  - Canvas画布（400x400像素）
  - 分数显示（当前得分、最高分）
  - 速度显示（当前选择的速度档位）
  - 控制按钮（开始、暂停、重置）
  - 游戏说明

### 3. 游戏核心逻辑

- **蛇的移动**：使用方向键控制（↑↓←→）
- **食物系统**：
  - 初始位置随机生成
  - 吃到食物增加10分和蛇的长度
- **碰撞检测**：
  - 撞墙：游戏结束
  - 撞自己：游戏结束
  - 撞障碍物：游戏结束

### 4. 障碍物系统（游戏进行中）

- **触发条件**：从第6个食物开始出现
- **生成规则**：
  - 第6-10个食物：每次刷新1个障碍物
  - 第11-15个食物：每次刷新2个障碍物
  - 以此类推，每5个食物增加1个障碍物
- **生成位置**：
  - 障碍物生成在蛇头3x3网格之外
  - 使用BFS算法确保玩家有路径可以吃到食物
  - 避免出现完全无法获取食物的死局
- **视觉效果**：灰色方块

### 5. 特殊食物系统

- **触发条件**：从第5个食物开始，1/8概率生成
- **特殊食物类型**：
  - 🔴 红色特殊食物：加速10秒（速度变为当前速度的1/2）
  - 🔵 蓝色特殊食物：减速10秒（速度变为当前速度的2倍）
- **视觉效果**：不同颜色区分不同功能

### 6. 游戏控制

- **开始游戏**：点击后开始游戏（需先选择速度）
- **暂停/继续**：暂停或继续当前游戏
- **重置游戏**：重置到初始状态，可以重新选择速度

### 7. 界面样式

- **整体风格**：现代简约，渐变背景
- **颜色方案**：
  - 蛇头：深绿色（#38a169）
  - 蛇身：绿色（#48bb78）
  - 食物：红色（#f56565）
  - 特殊食物：红色（加速）、蓝色（减速）
  - 障碍物：灰色（#718096）
  - 画布背景：深灰色（#2d3748）
- **按钮样式**：圆角、阴影效果，hover状态有动画

### 2. 核心算法

- **BFS路径检测**：用于确保障碍物生成后玩家仍能吃到食物
- **碰撞检测**：检测蛇与墙壁、自身、障碍物的碰撞
- **随机生成**：食物和障碍物的随机位置生成

### 3. 代码规范

- 变量命名清晰，遵循驼峰命名法

### 4. 事件绑定

- 使用document.getElementById获取元素
- 避免重复事件绑定
