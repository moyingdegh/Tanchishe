# AI 开发强制规则

## 1. 工作流程（必须严格遵守）
1. 每次开始编码前，**必须先完整阅读项目 README.md**
2. 严格按照 README 中的项目介绍、功能规范、代码规范开发
3. 禁止编写与 README 描述不符的代码
4. 任何代码修改，**必须同步更新 README.md**，保持文档与代码一致

## 2. 开发要求
- 变量命名、文件结构、功能逻辑必须匹配 README
- 新增功能 → 立刻补充到 README
- 删除/修改功能 → 立刻更新 README 对应说明
- 修复bug → 在 README 更新日志中记录

## 3. 代码质量要求
- 遵循现有代码风格和命名规范
- 保持代码简洁明了，避免过度设计
- 确保代码的可读性和可维护性

## 4. 文档更新要求
- README.md 是项目的唯一权威文档
- 所有变更必须及时更新到 README.md
- 更新日志要清晰记录版本变更内容
